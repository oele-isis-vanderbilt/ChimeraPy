Orchestracting class for running multiple data pipelines.

The GroupRunner inherents and thereby uses plenty of methods from SingleRunner. The main difference is that GroupRunner manages the processing of multiple SingleRunner and then an additional group data pipeline. The output of each SingleRunner can then be used as the input to the group data pipeline.
